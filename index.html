        Regex Filter

        Use the filter bar in the Log Viewer to search and highlight log entries. Supports full regex syntax.

        Failed passwordExact text match
        Failed|AcceptedMatch either word
        sshd\[\d+\]Match sshd with any PID
        45\.33\.32\.\d+Match IP range 45.33.32.*
        03:1[4-8]:\d{2}Match time range 03:14 – 03:18
        (?i)error|warn|critCase-insensitive severity match

        Detection Engine

        Three detection layers run on your loaded logs. Each layer adds depth to the analysis.

        Layer 1 (Rules)Signature/regex matches — appear instantly on load, tagged RULE
        Layer 2 (Statistical)Baselines, frequency spikes, off-hours, event chains — appear on load, tagged STAT
        Layer 3 (AI)Optional LLM — natural-language queries, summaries, incident reports, and behavioral analysis. On-demand, tagged AI

        Click any alert in the Detection Results panel to jump to the relevant log lines and see the enrichment details (IoC, MITRE ATT&CK, mitigation).

        Detection Engine

        Out of the box — these 3 layers run fully offline without any configuration. No API key, no internet, no setup.

        ON
        Layer 1 — Rule-based Regex pattern matching against known attack signatures: SQLi, XSS, path traversal, brute force patterns, web shells, and more. Instant results.
        Brute force SSH SQLi in URL Web shell access Path traversal
        ON
        Layer 2 — Statistical Builds a baseline from your logs, then flags deviations: frequency spikes, unusual hours, abnormal volumes, and event chains within time windows.
        Frequency spike Off-hours activity Volume anomaly Event chain
        OFF
        Layer 3 — AI Optional LLM layer — off until you connect a provider below. Activating AI adds: natural-language queries over your logs, auto-summarization of findings, incident report generation, contextual investigation hints, and behavioral analysis (e.g. beaconing / C2 patterns and sequence anomalies that the rule and statistical layers don't cover). All data is pseudonymized before sending and the API key stays in your browser.
        NL query Auto summary Incident report Hints Behavioral analysis

        AI Enhancement

        Optional — connect an AI provider to unlock Layer 3 (AI). Enables natural language queries, auto-summarization, incident report generation, and behavioral analysis. All data is pseudonymized before sending.

        ⓘ

        Pseudonymization Active

        All identifiable data (IP addresses, hostnames, usernames, emails, paths) is automatically replaced with consistent dummy values before being sent to the AI provider. Real data never leaves your browser.

        AI receives only summary statistics, not log content (even pseudonymized).
